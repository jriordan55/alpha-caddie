# AlphaCaddie (Java)

Spring Boot **3.4** + **Thymeleaf** + **WebClient**. All server logic is **Java** — no R runtime.

- **Model O/U**: merges DataGolf `field-updates` + `preds/skill-ratings`, imputes count means (same idea as R raw projections), prices lines with Normal / Poisson + juiced Americans (`OuMath`).
- **Matchups / Outrights**: fetches JSON and renders it (extend with table builders in Java as needed).
- **+EV / Hole Hangout**: stubs — implement in Java (CSV with OpenCSV / Jackson, MC in plain Java).

## Requirements

- JDK **17+** (Spring Boot 3; Java 8 is not enough)
- **Maven** — optional if you use the scripts below
- DataGolf API key: `DATAGOLF_API_KEY` or `datagolf.api-key` in `application.properties`

## Run (Windows — no global Maven)

**Easiest (works when scripts are disabled):** use the batch launcher from **cmd** or PowerShell:

```powershell
cd C:\Users\student\Documents\golfModel\java\alphacaddie
set DATAGOLF_API_KEY=your_key
run.cmd spring-boot:run
```

In PowerShell you can use `$env:DATAGOLF_API_KEY="your_key"` then `.\run.cmd spring-boot:run`.

First run downloads Maven into `.mvn-tools\` (gitignored).

**If execution policy allows scripts**, you can run `.\run.ps1 spring-boot:run` directly.

**One-liner without changing policy:**

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\Users\student\Documents\golfModel\java\alphacaddie\run.ps1" spring-boot:run
```

**Optional — allow local scripts for your user only:**

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

**Or** use the Maven Wrapper (needs `JAVA_HOME` set to JDK 17+):

```powershell
$env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-17.0.13.11-hotspot"   # your path
$env:DATAGOLF_API_KEY = "your_key"
.\mvnw.cmd spring-boot:run
```

## Run (if `mvn` is installed)

```bash
cd java/alphacaddie
export DATAGOLF_API_KEY=your_key   # Linux/macOS
mvn spring-boot:run
```

Open **http://localhost:8080**

## Build JAR

```bash
mvn -q package
java -jar target/alphacaddie-1.0.0-SNAPSHOT.jar
```

## Note on “Java” vs browsers

The **UI** is HTML generated by **Thymeleaf** (templates under `src/main/resources/templates`). Browsers still render HTML/CSS; there is **no** R, **no** Node, **no** Python in this module.
